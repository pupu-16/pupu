# Emotion Music Companion

## 触发场景

当任务涉及背景音乐、情绪音乐、环境声、播放体验、音乐与粒子生命体联动、睡前/专注/低落陪伴音乐时使用本 Skill。典型请求包括：

- 设计 AI 朋友如何根据聊天和日记推荐音乐。
- 设计 Claudio FM 式的氛围音乐体验。
- 规划背景音乐、环境声、情绪模式、音量和自动播放规则。
- 让主界面的粒子生命体随音乐轻微律动。

## 核心原则

- **音乐是陪伴，不是打扰**：默认低存在感，用户随时可暂停、静音、切换。
- **先征求，再播放**：禁止未经同意突然出声，尤其是首次进入应用。
- **情绪匹配要温柔**：低落时不一定推“悲伤音乐”，可能需要稳定、温暖、低刺激。
- **从状态到场景**：音乐模式应围绕用户场景，如入睡、写日记、陪伴、专注、散步、发呆。
- **声音与视觉统一**：音乐节奏、粒子生命体、界面光感应形成一个整体。
- **给用户控制感**：允许调节能量、温度、节奏、人声/无声、循环时间。

## 开发规范

- 设计音乐模式而不是简单歌单：calm、soft-focus、night、warm-lonely、tiny-joy、rain-room。
- 记录音乐偏好时需区分：喜欢的风格、讨厌的声音、触发不适的声音、睡眠禁忌。
- 支持环境声混合：雨声、房间噪声、磁带底噪、远处钢琴、低频 pad。
- 音乐推荐应可解释：说明“我选了更慢、更暖的声音”，不要假装读心。
- 粒子联动只使用柔和参数：脉冲、扩散、亮度、漂移速度，避免频谱秀。
- 默认提供静音优雅体验：没有音乐时 UI 仍完整。
- 音频资源、版权和来源必须明确，不使用未经授权素材。

## 示例

### 示例 1：情绪到音乐映射

```text
焦虑：低节奏、少旋律、稳定循环、无突兀鼓点
孤独：温暖质感、轻微空间感、低密度旋律
疲惫：更慢、更暗、更少变化，适合闭眼
想创作：轻节拍、明亮但不兴奋、可长时间循环
开心：轻盈、短循环、粒子更活跃，但不过度刺激
```

## 禁止事项

- 禁止首次进入应用自动播放声音。
- 禁止把用户情绪粗暴绑定到固定音乐，例如“悲伤就播放悲伤歌”。
- 禁止使用侵入式音效、频繁提示音或无法关闭的循环。
- 禁止使用版权不明的音乐素材。
- 禁止让音乐功能压过聊天、日记和陪伴本身。
- 禁止在用户明确需要安静时继续推荐声音。
