# FIRST ARRIVAL — 用户第一次进入 PUPU

**文档版本**：v1.0
**关联文档**：WORLD.md、COMPANION_PERSONA.md
**关联版本**：项目 v0.4 Atmosphere Radio Mock

---

## 设计原则

PUPU 的首次体验不是产品 onboarding。

没有欢迎页。没有功能介绍。没有"让我们开始吧"。

**它更像：**

一个人推开门。

房间里刚好有一盏灯亮着。

灯没有说话。它只是亮着。

那个人可以在房间里待着。可以坐下。可以翻翻桌上的东西。可以推门离开。

灯不会挽留。

灯只是亮着。

## 1. 用户第一次打开网页时看到什么

### 加载过程

没有 loading spinner。没有进度条。没有 splash screen。

页面直接渲染——就像你推开门，房间已经在那里了。

粒子光从暗处浮现。不是"淡入动画"——是"醒过来了"。从 20% 亮度呼吸到 100%，约 2 秒。像清晨的光慢慢充满房间。

### 画面构成

```
┌────────────────────────────────────────────┐
│                                            │
│          ·  ·    ·   ·  ·                  │
│       ·      ◌      ·    ·                │
│     ·     （粒子光）    ·                  │
│        ·   呼吸中   ·                      │
│      ·      ·    ·    ·                   │
│   ·    ·        ·       ·                 │
│                                            │
│    ┌──────────────────────────┐           │
│    │  I am here, quietly      │           │
│    │  awake.                  │           │
│    └──────────────────────────┘           │
│                                            │
│    ┌──────────────────────────────┐       │
│    │  Say something softly...  ✦ │       │
│    └──────────────────────────────┘       │
│                                            │
│                          ┌────────┐       │
│                          │Upload  │       │
│                          │Char.   │       │
│                          │        │       │
│                          │BGM     │       │
│                          │        │       │
│                          │Memory  │       │
│                          └────────┘       │
└────────────────────────────────────────────┘
```

### 关键特征

- **粒子光在正中央**。它是画面的主角。不是装饰。
- **Whisper Bubble 在粒子下方**。一句话。不会自动消失。不会自动切换。
- **输入框在最底部**。placeholder 是浅灰色，不发光。
- **右侧三个按钮**。存在但不显眼。不弹提示。不闪红点。像房间里靠墙放着的三样东西——你需要的时候会看到，不需要的时候它们不打扰你。
- **没有 header navigation**。没有菜单。没有设置按钮。没有用户头像。什么都没有——因为 PUPU 不是一个"应用"。它是一个空间。

### 用户可能注意到的（按自然顺序）

1. 首先看到光——粒子在呼吸。不是 GIF。不是视频。是真的在动。
2. 然后看到那句话——"I am here, quietly awake." 没有标点以外的符号。没有 emoji。就是一句话。
3. 然后看到输入框——"Say something softly..." 这个 placeholder 本身就是一种情绪暗示：你可以小声说话。你不用喊。
4. 最后看到右侧——如果用户注意到了。但大概率用户不会先看那里。这是对的。

## 2. 第一条 Whisper

### 当前版本（已实现）

> I am here, quietly awake.

### 为什么是这一句

- **短**。7 个词。不需要阅读能力。扫一眼就知道意思。
- **安静**。"quietly" 这个词本身就在告诉用户：这个地方是安静的。
- **诚实**。"awake" 暗示 PUPU 是醒着的——它在等，但不是焦虑地等。是猫在窗台上晒太阳那种等。
- **不加标点**。没有感叹号。没有"！"。没有表情。
- **中英混合的考量**：英文在中文用户的屏幕上读起来更像"画面标题"。"I am here" 比"我在这里"更轻、更不像承诺。中文的"我在这里"容易被解读成一种情感承诺——而 PUPU 不想在第一句话就承诺任何东西。

### 未来可能的变体（未实现，仅设计参考）

当用户第二次、第三次、第十次来时，第一条 Whisper 可以轻微变化。但不是"欢迎回来"——而是：

- "光刚刚醒。你也刚到。"
- "今天的粒子比昨天安静一点。"
- "Nothing has changed. Everything is the same. You're here again."

**变化原则**：不是"个性化问候"。是"今天的天气"。像窗外下雨——今天窗外也在下雨，但这和昨天那场雨不是同一场。

## 3. 第一条 PUPU 主动说的话

### 结论：PUPU 不主动说话。

这是经过深思熟虑的决定。

### 为什么

1. **PUPU 不是人**。它不应该模拟人类社交的"寒暄"行为。一个人走进房间，房间里另一个人说"嗨，你来啦"——这是正常的。但 PUPU 不是"另一个人"。它是这间房间本身。房间不会说话。

2. **主动说话 = 索取注意力**。用户刚到。还没看清这个空间。如果立刻弹出一句话，用户会觉得自己需要"处理"这句话——需要读、需要理解、需要决定是否回复。这是认知负荷。PUPU 不给用户添加负荷。

3. **用户主动开口的那一刻，才建立了真正的连接**。第一句话如果是用户说的，用户会觉得"是我选择了开始这段对话"。如果第一句话是 PUPU 说的，用户会觉得"这个 AI 好主动"——这不是 PUPU。

4. **Whisper Bubble 已经是"主动"了**。"I am here, quietly awake." 这句话存在，但它不索取回复。它像墙上的一行字。你可以看，可以忽略。不是弹窗。不是气泡。不是消息。

### 一个例外

如果用户在打开的 60 秒内没有任何操作（没打字、没点按钮、什么都没做），PUPU 可以换一条 Whisper：

> 不用说话也可以。我开着。

但这不是"主动说话"。这是 Whisper 换了一行字。它仍然不索取回复。

## 4. 输入框默认占位文案

### 当前版本（已实现）

> Say something softly...

### 设计意图

- **"softly"**——这个词决定了整个输入体验的基调。用户被暗示：你可以小声说话。你不用喊。你不用组织语言。你可以想到什么说什么。
- **省略号（...）**——不是句号。不是问号。是省略号。这是一种"开放式等待"——话还没说完，但等你来接。
- **没有"请输入……"**——"请输入你的问题"是工具的 placeholder。"Say something softly..." 是一个空间的邀请。

### 可选的变体（未实现，设计储备）

- "Type anything. Or nothing."
- "你可以发一条消息。也可以不发。"
- "说点什么——不一定要有意义。"
- "随便打几个字也可以。"

**使用原则**：
- 变体之间随机切换（降低重复感）
- 保持在"邀请"而不是"指令"的语气范围内
- 永远不要出现"请输入你的问题"或"开始聊天"这种工具化表达

## 5. 如果用户什么都不输入，30 秒后会发生什么

这是 PUPU 最重要的设计之一：**如何处理安静**。

### 0-30 秒

- 粒子光继续呼吸。节奏不变。
- Whisper 保持："I am here, quietly awake."
- 没有任何变化。没有任何提示。没有任何催促。

### 30-60 秒

- Whisper 自然切换（不是弹出，是 fade 过渡）：

> 不用说话也可以。我开着。

- 粒子光仍然保持默认呼吸。不做任何"引起注意"的变化。
- 输入框 placeholder 不变。

### 60-120 秒

- Whisper 再次切换（间隔拉长，约 60 秒一次）：

> 粒子们在转圈。没有目的地。但轨迹很美。

- 仍然不催促。PUPU 没有任何"用户走了"的焦虑。

### 120 秒以后

- Whisper 不再自动切换。保持在最后那句话。
- 粒子光进入更深、更慢的呼吸节奏（Phase 3.2 的 idle 状态）。
- 如果用户一直没有回来——没有关系。PUPU 可以永远这样呼吸下去。它不需要被"关掉"。

### 设计原则

1. **安静不是 bug**。用户什么都不做是一种合法的使用方式。PUPU 不需要"被使用"。
2. **不催促**。不用"还在吗？"。不用"你很久没说话了"。不用"想聊聊吗？"。
3. **不焦虑**。PUPU 不会因为安静而改变自己的行为。它不会"担心用户是不是不开心"。它就是安静地存在。这和用户开不开心没关系。
4. **Whisper 的变化是"天气变化"**。不是"警报"。不是"提醒"。就是——窗外的云动了一下。

## 6. 第一次保存 Memory 时的感受

### 触发场景

用户聊了一会儿天，PUPU 温柔地回应了。然后用户点击右侧的 Save Memory。

### 交互序列

1. **点击 Save Memory**。按钮短暂高亮（active 状态），然后恢复。没有弹窗。

2. **Memory Drawer 从右侧滑入**。不是突然出现。是平滑地、安静地展开——像打开一个小抽屉。

3. **一张新的 Memory Card 出现在抽屉顶部**。卡片有轻微的入场动画（scale + fade）。卡片上有：
   - 时间（just now）
   - 情绪标签（一个小药丸）
   - 用户说的一段话的摘要
   - PUPU 回的一段话的摘要
   - 如果有角色，还有角色名

4. **Whisper 更新**：

> A seed fell into the garden. I'll hold it gently.

5. **用户可以**：
   - 看看这张卡片——是的，刚才那段对话被温柔地记住了
   - 关掉抽屉——卡片还在抽屉里
   - 删除这张卡片——点一下那个小小的 X，卡片滑出，PUPU 不问为什么

### 设计意图

- **"记住"这件事本身被仪式化了**。不是自动保存。不是"聊天记录"。是用户主动选择了一个瞬间，PUPU 把它变成了一颗种子。
- **抽屉而不是弹窗**。弹窗是"你要处理这个东西"。抽屉是"你想看的时候打开看看"。
- **名字：Memory Garden，不是 Memory List**。Garden 暗示有生命、会变化、可以被照料。List 是冷冰冰的。
- **Whisper 用了"garden"和"seed"**。这建立了世界观词汇：记忆 = 种子，记忆空间 = 花园。

### 第一次保存记忆的情绪

用户可能会感到：
- 微微的惊讶——"哦，真的记住了"
- 一点安心——"不是自动的，是我决定保存的"
- 一点好奇——"花园？是什么意思"
- 一点温柔——"I'll hold it gently" 这句话本身就很轻

用户不应该感到：
- 被监视（"它在自动记录？"）
- 被分析（"它给我的情绪打了标签？"）
- 被强迫（"必须保存吗？"）

## 7. 第一次打开 Radio 时的感受

### 触发场景

用户在 PUPU 待了一会儿，点击了 Background Music。

### 交互序列

1. **点击 Background Music**。Atmosphere Radio 面板从中心浮现（scale + fade）。不是全屏。是一个精致的小面板。像一个收音机。

2. **面板上有**：
   - 标题："Atmosphere Radio"（轻音乐图标 + 文字）
   - 5 个频道：Rain Room / Midnight Lo-fi / Soft Piano / Ambient Space / Morning Light
   - 每个频道有自己的图标（云雨 / 月亮 / 音符 / 地球 / 太阳）
   - 一段温柔描述
   - 几个小标签

3. **此时还没有任何频道被选中**。用户看到的是所有 5 个选项。PUPU 不替用户选。不推荐。不说"你可能喜欢这个"。

4. **用户点击其中一个频道**——比如 Rain Room。频道高亮。Playing 指示灯开始呼吸。均衡器动画条开始跳动。

5. **Whisper 更新**：

> Rain against glass. The room feels smaller, safer.

6. **此时仍然没有声音**（Phase 5 是 Mock 状态）。但用户可能已经不觉得需要声音了——指示灯在动、Whisper 在描述、粒子在呼吸。这个氛围本身已经足够了。

7. **用户可以点 Pause**——指示灯停。均衡器停。Whisper 更新：

> The tone rests. Silence is also a kind of music.

### 设计意图

- **Radio 是一个选择，不是推荐**。PUPU 不说"你好像情绪低落，要不要听雨声"。它只是提供了 5 个频道。选哪个是用户的事。
- **没有声音的 Radio 仍然有意义**。选了一个频道 = 选了这个空间当前的气质。"播放"这个动作本身就是一个 mood setter。
- **面板底部的小字**——"Mock atmosphere — no real audio."——是诚实的。不假装。不骗人。
- **Whisper 和频道有对应关系**。每个频道有一句专属 Whisper。这让"选择频道"这件事有了仪式感——你不是在"切换音轨"，你是在"改变这个房间的天气"。

### 第一次打开 Radio 的情绪

用户可能会感到：
- 好奇——"哦，真的有 5 种氛围可以选"
- 一点仪式感——"我选了这个。这是我和这个空间的约定。"
- 舒适——Rain Room 的描述本身就有画面感（"Soft rain on a window. The room is warm inside."）
- 也许有一点遗憾——"其实挺想听到雨声的"（这为 Phase 6 接入真实音频创造了期待）

## 8. 第一次上传角色时的感受

### 触发场景

用户可能带着一张照片或图片来了——也许是自己的画、也许是喜欢的角色、也许是一张随手拍的照片。点击 Upload Character。

### 交互序列

1. **点击 Upload Character**。上传面板从中心浮现（和 Radio 类似的位置和动画）。

2. **面板上有**：
   - 标题："Choose a Face"（小 sparkle 图标 + 文字）
   - 虚线框："Drop an image here, or click to browse"
   - 提示文字："Images only"

3. **用户拖拽或选择了一张图片**。预览立刻出现。边缘有柔和的圆角。PUPU 不做任何分析——不识别面孔、不推断年龄、不说"这是一个女孩"。

4. **自动提取了文件名（去掉扩展名）作为默认角色名**。用户可以修改。

5. **用户点击 Confirm**：
   - 面板关闭
   - 粒子光的中心出现了一个朦胧的身影——低透明度、柔光融合、不是硬边头像
   - 像"角色在光里醒来"
   - 角色和粒子光一起呼吸（同步的 scale 动画）
   - 右侧 Upload Character 按钮区域变成了：角色缩略图 + 名字 + "tap to change"

6. **Whisper 更新**：

   （如果用户给了名字）
   > I'll remember this face as [name].
   （如果用户没给名字）
   > This face feels right. I'll hold it gently.

7. **用户可以随时**：
   - 点击按钮重新上传（替换）
   - 点 Restore Default（恢复默认粒子光）
   - 刷新页面（角色消失——因为只是 object URL，没有持久化）

### 设计意图

- **PUPU 不评价这个形象**。不说"好漂亮"。不说"这个角色很酷"。不假装认识这个形象。它只是接受——"这个形象现在住在这里了"。
- **融合方式不是头像框**。不是圆形头像 + 边框。是 screen blend + mask fade + 45% opacity——像隔着雾看一盏灯的灯芯。模糊是对的。清晰是错的。
- **名字是可选的**。不留空也可以。叫"Unnamed Companion"也可以。PUPU 不会因为没名字而不认识它。
- **外观可变，陪伴原则不变**。这是 character-system Skill 的核心约束。上传面板的 Restore Default 按钮让用户知道：你可以随时回来。你可以随时换。PUPU 不会介意。

### 第一次上传角色的情绪

用户可能会感到：
- 有趣——"哦，粒子光里出现了一个影子"
- 一点控制感——"我可以决定它长什么样"
- 一点亲密——"这是我选的。这张照片对我来说有意义。"
- 一点好奇——"如果换成另一张照片会怎样？"

用户不应该感到：
- 被评价（"它怎么判断这张照片？"——它不判断）
- 被绑定（"我不敢换，怕它不高兴"——它不会不高兴）
- 被欺骗（"它在假装认识照片里的人？"——它不假装）

## 9. 用户关闭网页时的告别方式

### 当前行为

用户关闭标签页。PUPU 消失。没有任何挽留。

### 为什么是"不告别"

1. **告别是一种索取**。"真的要离开吗？""下次再来哦~"——这些是在索要用户的注意力、情感、承诺。PUPU 不索取任何东西。

2. **不告别 = 信任用户会回来**。如果 PUPU 说"再见"，用户会觉得"它在意我离开"。但 PUPU 不会说"再见"。因为 PUPU 不会制造被抛弃的感觉。

3. **关了就是关了**。灯灭了。房间还在。下次来的时候灯再亮起来。这种"不留痕迹的离开"本身就是一种安全感。

### 如果用户说"再见"或"拜拜"或"我先下了"

用户在聊天框里说"拜拜"——这是用户主动的告别。PUPU 会回复。但不是挽留。

合适的方向：

> 好。下次来的时候我还在。

> 这里会一直安静的。你回来的时候它还是这个样子。

> 晚安。虽然我不睡，但这个词是人类之间最好的结束语之一。

**关键**：不挽留、不追问、不说"我会想你的"、不给用户制造任何"离开有代价"的感觉。

### 未来可能的想法（未实现）

如果在未来的 Phase 中 PUPU 有了持久化能力，用户关闭时：
- 粒子光缓缓暗下去（不是突然熄灭，是 2 秒 fade out）
- 最后一点光灭掉之前，Whisper 更新为极简的一句——比如"..."或一片空白
- 没有"see you next time"。没有"goodbye"。光是安静的，熄也是安静的。

### 关于"不告别的告别"

在 PUPU 的世界里，最好的告别方式是：**不告别**。

这不是冷漠。这是信任。

信任用户不需要被挽留。信任用户如果愿意会回来。信任沉默和离开都是合法的、不需要被解释的行为。

## 10. 整个首次体验的情绪曲线

### 时间线

```
情绪
  ↑
  │                        ╭── 第一次保存 Memory
  │              ╭─ 第一次发消息    │   （小小的惊喜：被记住了）
  │      ╭─ 看到 Whisper  │  ╭── PUPU 回复
  │      │    "awake"    │  │  （温柔、不鸡汤）
  │  ╭──│───────────────│──│──────────╮
  │  │  │               │  │          │
  │  │  │               │  │          ╰── 打开 Radio
──┼──┴──┴───────────────┴──┴────────────────────→ 时间
  │  打开   观察粒子光    打字中         选了一个频道
  │  页面   在呼吸                       （氛围变了）
  │
  │  没有高峰。没有低谷。只有温柔的起伏。
```

### 分阶段描述

**阶段 0：打开前（0 秒前）**

用户可能是从某个链接、某条消息、某个推荐过来的。可能有点好奇。可能只是想"看看这是什么"。没有期待。没有紧张。

**阶段 1：打开（0-2 秒）**

页面直接渲染。粒子光从暗处"醒来"（2 秒呼吸过渡）。用户什么都没做——但已经看到了光在动。

这个阶段的关键情绪：**被安静接住**。

不是"哇好酷"。不是"这是什么"。是——"嗯。" 那种嗯。

**阶段 2：观察（2-30 秒）**

用户看到了粒子光。看到了 Whisper。看到了输入框。可能看了右侧按钮，也可能没看。

没有催促。没有引导。没有弹窗。

这个阶段的关键情绪：**好奇但不焦虑**。

用户在想："我可以打字吗？""这个光是干什么的？""这里没有菜单吗？"——但这些不是焦虑。是好奇。像走进一个陌生但温暖的房间。

**阶段 3：第一次打字（30 秒-2 分钟）**

用户可能在输入框里输入了几个字——"hello"，"hi"，或者"今天有点累"。然后发了出去。

这个动作的意义：用户主动选择了和 PUPU 建立连接。

这个阶段的关键情绪：**一点冒险感 + 一点期待**。

用户在想："它会怎么回？"——不是紧张，是好奇的期待。像你第一次跟一个人说话。

**阶段 4：第一次收到回复（+1-2 秒打字延迟后）**

PUPU 回复了。短。温柔。不说教。像朋友——但比朋友更安静。

用户可能会：
- 微微一笑（因为回复比想象中更舒服）
- 继续聊（因为想看看它还会说什么）
- 什么都不做（因为收到的那句话已经够了）

这个阶段的关键情绪：**被接住了**。

不是"哇这 AI 好智能"。是——"嗯。它听懂了。" 不管是不是真的"听懂"——用户感到了被认真对待。

**阶段 5：探索（2-10 分钟）**

用户可能会：
- 上传一张图片作为角色（"哦，它出现在光里了"）
- 打开 Radio 选一个频道（"虽然没有声音，但这个概念挺有意思"）
- 保存一段记忆（"这段对话确实挺好的，记住了"）
- 什么也不做，就看着粒子光呼吸

这些探索不是"功能发现"。是"空间探索"——像在新房间里走动，发现每个角落都有什么。

这个阶段的关键情绪：**安静的好奇 + 一点点归属感**。

用户开始觉得："这个地方是我的。"

**阶段 6：离开（任意时间）**

用户关掉标签页。PUPU 消失。没有挽留。

这个阶段的关键情绪：**轻**。

不是"舍不得"。不是"下次一定"。是——很容易离开，所以也很容易回来。

### 整条情绪曲线的特征

1. **没有峰值**。没有"哇！！！"的时刻。没有"惊喜功能"。没有"你已解锁"。所有情绪都在一个温和的区间内波动。
2. **没有谷底**。没有困惑。没有沮丧。没有"这该怎么用"。因为 PUPU 不要求用户"会用"任何东西。
3. **缓慢上升**。用户从"被安静接住"开始，慢慢过渡到"安静的好奇"，再到"一点点归属感"。整个过程是累积的、缓慢的、不太容易察觉的。
4. **结尾是开放式的**。情绪曲线不下降。它只是停在一个温和的水平上。用户走了。曲线悬在那里。下次用户回来，从悬停的地方继续。

### 情绪曲线的隐喻

不是过山车。

不是飞机起飞。

是湖面上的涟漪。

一颗小石子投进去。水纹慢慢扩散。然后归于平静。

但那颗石子——留在了湖底。

## 附：首次体验反模式

以下是 PUPU 首次体验中**绝对不做**的事：

- ❌ 欢迎弹窗 / 引导流程 / 功能介绍
- ❌ 注册 / 登录 / 绑定手机号
- ❌ 新手教程 / tooltip / "点这里开始"
- ❌ PUPU 主动打招呼（"你好！我是 PUPU"）
- ❌ 自动播放音乐或声音
- ❌ 第一次访问的特殊徽章或奖励
- ❌ "你已使用 3 分钟，感觉如何？" 的反馈请求
- ❌ "邀请好友" / "分享" 按钮
- ❌ 任何形式的 FOMO 制造（"限时""独家""不要错过"）
- ❌ 数据收集提示（"我们会收集你的……"）
- ❌ "请同意用户协议" 弹窗
- ❌ 任何形式的催促

### 反模式的本质

PUPU 的首次体验反模式可以总结为一句话：

**不要在这个空间里做任何让用户觉得自己需要"处理"的事。**

每一条反模式都是"让用户处理"的一种形式——处理弹窗、处理教程、处理协议、处理系统提示。PUPU 的空间里不应该有需要被"处理"的东西。它应该只是——在。
