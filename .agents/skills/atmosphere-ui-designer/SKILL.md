# Atmosphere UI Designer

## 触发场景

当任务涉及主界面、视觉风格、粒子生命体、情绪空间、动效、沉浸感、布局、配色、字体、交互氛围时使用本 Skill。典型请求包括：

- 设计主界面为可交互粒子生命体。
- 设计类似 Claudio FM、Memory Garden、数字宠物、沉浸式情绪空间的审美系统。
- 规划聊天、日记、音乐、记忆花园如何融入同一个氛围界面。
- 审查 UI 是否过于普通、卡片化、工具化或缺少情绪质感。

## 核心原则

- **氛围是第一界面**：用户进入应用先感到“被安静接住”，再看到功能。
- **生命体不是装饰**：粒子生命体应响应聊天、音乐、情绪、记忆和触摸，而不是背景特效。
- **少即是深**：减少信息噪音，用空间、光、运动和声音暗示状态。
- **情绪可见但不标签化**：避免把用户情绪粗暴显示成“悲伤 72%”，更像天气、光晕、节奏变化。
- **高审美不等于堆特效**：动效要克制、连贯、有呼吸感。
- **功能藏在空间里**：聊天、日记、记忆、音乐可进入，但不应把主界面变成后台面板。

## 开发规范

- 主视觉优先定义：粒子生命体的形态、待机呼吸、靠近、回应、记忆闪烁、音乐律动。
- 建立情绪视觉参数：速度、密度、亮度、颜色温度、扩散半径、粒子聚合度。
- 使用暗色或柔和低饱和基底时，要通过局部高光和清晰层级避免沉闷。
- UI 控件应轻量、半隐藏、可呼出；不要在主屏堆满按钮和说明文字。
- 聊天层可以像字幕、耳语、漂浮消息或底部抽屉，但必须保证可读性。
- 音乐状态可用波纹、粒子脉冲或环境光表现，不依赖大段文字解释。
- 日记和记忆空间应有“收藏感”：像保存星尘、种子、光点，而不是普通列表。
- 所有动效必须考虑低性能设备、减少动态效果偏好和可访问性。

## 示例

### 示例 1：粒子生命体状态

```text
idle: 慢速呼吸，粒子微聚合，中心有柔和心跳
listening: 粒子向用户输入方向轻微偏移，亮度降低，像认真靠近
responding: 中心扩散细小波纹，伴随文字出现
music: 粒子按节拍轻微脉动，不做刺眼频闪
memory: 远处浮现小光点，表示某段记忆被温柔唤起
```

### 示例 2：主界面构成

```text
背景：沉浸式渐变/噪声/深色空气感
中心：可交互粒子生命体
底部：极简输入区
侧边：日记、音乐、记忆入口，默认收起
状态：通过光、运动、微文案表达，不用密集数据面板
```

### 示例 3：审美判断

合适：像一个会呼吸的情绪空间。

不合适：像普通 SaaS dashboard 换了深色背景。

## 禁止事项

- 禁止用大量卡片、营销式 hero、模板化渐变块替代真实体验。
- 禁止让粒子生命体只是无意义背景，必须和用户行为有关。
- 禁止 UI 文字覆盖、拥挤、难读或在移动端溢出。
- 禁止强频闪、高对比闪烁或不可关闭的剧烈动效。
- 禁止把情绪体验做成冷冰冰的分数仪表盘。
- 禁止为了“高级感”牺牲可用性、输入效率和可访问性。
